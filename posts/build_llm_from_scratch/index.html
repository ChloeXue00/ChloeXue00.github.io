从 0 搭建一个 LLM 全流程#

阶段划分#

阶段 目标 产出物
1 数据准备 语料收集/清洗 预训练数据集
2 模型架构设计 Transformer模型构建 初始模型结构
3 预训练（PTM） 学会语言知识 基础大模型
4 SFT（指令微调） 学会理解和执行指令 SFT后模型
5 DPO / RLHF 优化为人喜欢的风格 对齐后的模型
6 部署优化 推理优化、上线 可用API/产品

1 数据准备#

语料收集#

  • Web data（CommonCrawl、Wikipedia、Books、Reddit、Arxiv、Github code 等）
  • 开源数据集（The Pile, C4, OSCAR, OpenWebText 等）
  • 自建爬虫抓取数据

清洗与处理#

  • 去噪（去广告、去低质量段落）
  • 去重（SHA去重）
  • 分词（如 sentencepiece, BPE）
  • 数据格式转换（jsonl, tsv）
  • 长度过滤（过短/过长段落丢弃）

产出物：Tokenized corpus，准备供模型训练。

2 模型架构设计#

核心：Transformer 解码器架构（GPT类模型）

推荐开源实现#

  • Huggingface Transformers
  • nanoGPT
  • Megatron-LM
  • GPT-NeoX / GPT-J

模型设计参数#

参数 典型值（GPT-3 类模型）
hidden_dim 12288
n_layers 96
n_heads 96
vocab_size 50k
max_position_embeddings 2048
activation GELU

如果从小模型做起，1亿～3亿参数是初学者可训练范围。

3 预训练（PTM阶段）#

训练目标#

Causal Language Modeling → 预测下一个token

Input: "The quick brown fox jumps over the lazy"
Target: "dog"

工具栈推荐#

  • Trainer: Huggingface Trainer / DeepSpeed / Megatron-LM Trainer
  • Optimizer: AdamW + CosineDecay LR
  • Distributed: DeepSpeed ZeRO Stage 2 or Stage 3
  • Mixed Precision: FP16 or BF16

训练配置示例#

batch_size: 1024
learning_rate: 1e-4
warmup_steps: 10000
total_steps: 500000
gradient_accumulation_steps: 8
fp16: true

注意：数据吞吐瓶颈是常见问题，DataLoader 优化很重要。

产出物：初步学会语言的大模型 Checkpoint。

4 SFT（指令微调）#

数据集#

  • FLAN / SuperNI

  • OpenAssistant / Dolly / ShareGPT

  • 自行构造（指令 + 回答对）

训练方式#

  • 监督学习：CrossEntropy loss

  • 普通 fine-tune 即可，不需要 RL

Prompt设计#

  • Instruction: Translate the following sentence into French.
  • Input: “Good morning.”
  • Response: “Bonjour.” 产出物：能理解指令的模型。

5 DPO / RLHF#

DPO#

  • 数据格式：prompt + positive_answer + negative_answer
  • 优化 loss：DPO loss（对比学习 loss）

RLHF#

  • 收集人类打分 → 训练 Reward Model

  • PPO 训练 base model

  • 工具：TRL PPOTrainer / RLHF RLlib 产出物：对齐人偏好的模型，正式上线版本。

6 部署优化#

Quantization#

  • GPTQ / AWQ / LLM.int8 → 把模型压缩到 INT8 / INT4 推理

  • 开源工具：AutoGPTQ, llama.cpp, exllama

Serving#

  • vLLM（高性能批推理）

  • TGI (Text Generation Inference, Huggingface)

  • llama.cpp（轻量 CPU/GPU 本地推理）

总结 Data Collection → Data Cleaning → Tokenization → PTM → SFT → DPO / RLHF → Deployment

工程化开源项目

项目名 特点
nanoGPT 极简 GPT 训练
GPT-NeoX 大规模 GPT 训练
Huggingface Transformers 最全生态
Huggingface TRL RLHF/DPO 支持
Megatron-LM 超大规模 GPT 支持
vLLM 高性能推理框架
llama.cpp 轻量 CPU/GPU 推理

  • Huggingface Transformers → 有大量预训练模型可以直接用

  • GPT-NeoX → 社区有预训练模型可以直接用（HF Hub 上 GPT-NeoX 20B 等）

  • Megatron-LM → 通常需要自己训练 or 企业内部使用，公开模型少

  • llama.cpp → 本身不负责训练，负责部署，加载 LLaMA / Qwen / Mistral 等现有预训练模型

  • nanoGPT → 教学用途，自己训练的小模型，一般不直接产品化用

  • TRL → 用已有模型进行 DPO / RLHF 优化，不负责训练 base model

  • vLLM → 用已有 Transformers 模型做高性能推理，不负责训练

  • 从0-1 搭建一个tiny大模型的问题：

  • 数据 pipeline 怎么做

  • fp16 / bf16 mixed precision 怎么踩过坑

  • checkpoint 怎么存，推理 latency 怎么评估

  • tiny model 的 scaling law 有啥观察

  • 过拟合现象怎么处理

  • training curve
